Example 1: implementation plan

phases
Step 01 Audit outputs Find places where the agent writes dense plans, reports, or reviews.
Step 02 Choose layout Plan, comparison, explainer, editor, or PR review surface.
Step 03 Add exports Copy prompt, JSON, diff, selected option, or follow-up checklist.
Step 04 Verify Open the file, check responsiveness, and confirm buttons work.
Plan signal HTML treatment Why it helps
Many phases Timeline lanes The reviewer sees sequence and scope instantly.
Risky decisions Tradeoff table Alternatives and consequences are not hidden in prose.
Next prompt needed Copy as prompt The artifact becomes part of the agent loop.

Example 3: architecture map

system
Client action User creates a plan, selects options, and exports decisions.
->
Agent artifact HTML stores readable structure plus machine-readable exports.
Boundary Contract Failure mode
Human review Readable visuals and explicit decisions Pretty page hides weak reasoning
Agent loop Export only meaningful changed values Copying the whole page adds noise
Sharing Self-contained local HTML Remote assets make it brittle
