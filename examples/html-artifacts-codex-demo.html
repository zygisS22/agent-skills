# Implementation Plan
## Context
We need to refactor the artifact generation flow so the agent can decide between Markdown and HTML output based on information density, review needs, export needs, and the user's likely next action...

## Phase 1
- Audit call sites that produce long-form plans.
- Identify outputs that include architecture, tables, diffs, diagrams, or interactive choices.
- Create a policy for when HTML is appropriate.
- Add verification so generated HTML is readable at mobile and desktop widths.

## Phase 2
- Build a comparison layout.
- Add export buttons.
- Include copy as prompt, copy as JSON, copy diff.
- Validate empty states and fallback behavior.

## Risks
- Too much styling can hide the actual decision.
- Remote assets can make the artifact brittle.
- Giant pages can become harder to diff than Markdown.
- Interactive state should export only meaningful choices.

## Acceptance Criteria
- Responsive layout.
- No overlapping controls.
- Source notes are present.
- Export actions are visible.
- The artifact can be shared as one file.
